What this app checks. IONOS runs the RHUI (Red Hat Update Infrastructure) servers that supply RHEL updates to your VMs, but this dashboard does not have direct access to that infrastructure. Instead, it uses SSH to log into a test client you provide below — a RHEL VM that is already registered to pull updates from IONOS RHUI — and runs the exact same checks a real update would run: reading its repo configuration, resolving and reaching the RHUI server, validating certificates, and attempting a live metadata fetch. The RHEL release is detected automatically once connected — you don't need to specify it. This shows you precisely what a customer's server experiences when it tries to update, and where in that chain a failure is happening.

Values are kept in server memory for this session unless you click "Save to .env".
